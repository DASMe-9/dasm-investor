# مقارنة: برنامج الأصدقاء vs تنفيذ DASM الحالي

> هذا الملف يجيب على سؤال: هل ما بُني في داسم مشتق من ملف الإكسل؟

---

## الإجابة المباشرة: **لا — التنفيذان مختلفان جوهرياً**

ما بُني في `AngelInvestmentSubscription` في داسم هو نظام عقود استثمار ملائكي عام (single-position, admin-managed). أما هذا الملف فهو برنامج تمويلي ذو شرائح وآلية توزيع متطورة. كلاهما يشتركان في فكرة "15% من دخل المنصة" لكنهما يختلفان في كل شيء آخر.

---

## جدول المقارنة التفصيلي

| المحور | برنامج الأصدقاء (الملف) | تنفيذ DASM الحالي |
|--------|------------------------|-------------------|
| **الشرائح** | صغار (< 100K) + كبار (≥ 100K) | شريحة واحدة لكل اشتراك |
| **وعاء الصغار** | 15% من **إجمالي الدخل** (Gross) | 15% من **صافي الدخل** (Net) |
| **وعاء الكبار** | نسبتهم من **صافي الربح** | لا يوجد تمييز |
| **السقف** | 2x كامل → 3x نصف → 4x ربع → ينتهي | cap_multiple واحد — ينتهي مباشرة |
| **عتبة التفعيل** | 150K تراكمي للمجموعة | Admin يفعّل يدوياً |
| **الترقي التلقائي** | نعم — عند 100K فردي | لا يوجد |
| **التخارج** | 3 سيناريوهات + 3 سنوات حظر + 3 شهور | لا يوجد |
| **نموذج الدخل** | مرتبط بعدد الصفقات (250/500/1000) | منفصل عن نموذج الدخل |
| **إدارة التوزيع** | تلقائي حسب النسبة التراكمية | Admin يسجّل يدوياً |
| **التدقيق القانوني** | نص عقد كامل موثّق | عقد وتوقيع إلكتروني |
| **حساب النسب** | نسبة تراكمية تتغير كل شهر | رأس مال ثابت وقت التفعيل |

## ما الأفضل في كل نموذج؟

### ✅ ما يتميز به الملف
- **عدالة أكبر:** النسبة التراكمية تكافئ من يستمر في الدعم
- **مرونة التخارج:** 3 سيناريوهات واضحة وموثّقة
- **تحفيز الاستمرار:** المضاعف المتناقص يمنع الانسحاب المبكر
- **شفافية نموذج الدخل:** الصفقات → العمولات → الخصومات → الصافي
- **التمييز العادل:** الكبار يأخذون من الصافي لا الإجمالي

### ✅ ما يتميز به تنفيذ DASM
- **قانوني ومحكم:** توقيع إلكتروني + تجزئة SHA256 + سجل تدقيق
- **مرن للأدمن:** إنشاء عقود، نسخ، تفعيل، إيقاف، إغلاق
- **مرتبط بالليدجر:** كل توزيع = قيد مالي موثّق
- **SSO:** وصول آمن عبر منصة داسم

## التوصية

### للمدى القصير — السيميوليشن
ابنِ السيميوليشن من هذا الملف كـ `/simulation` في `dasm-investor`:
- يساعد على فهم الأرقام بدون ضغط "إذا غلطت خربت البيانات"
- يُمكّن من اختبار سيناريوهات مختلفة (نمو أسرع / أبطأ، مستثمرون جدد)
- يُولّد ثقة قبل الربط الحقيقي

### للمدى المتوسط — تطوير DASM
يمكن دمج أفضل ما في الملف داخل DASM:
1. **إضافة نوع الشريحة** إلى `angel_investment_subscriptions` (صغار/كبار)
2. **إضافة المضاعف المتناقص** كـ phases (2x → 3x → 4x) بدل cap_multiple الواحد
3. **ربط وعاء الصغار** بإجمالي الدخل لا الصافي
4. **إضافة عتبة التفعيل** كـ config قابلة للتعديل

### ما يُحذف بعد الانتهاء
```
// في App.tsx — سطر واحد للحذف:
<Route path="/simulation" component={Simulation} />

// في Layout.tsx:
{ href: "/simulation", label: "السيميوليشن" }
```
