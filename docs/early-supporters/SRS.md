# SRS — برنامج أصدقاء الدعم المبكر
## وثيقة متطلبات البرنامج (Software Requirements Specification)

> **الإصدار:** 1.0  
> **المصدر:** ملف Excel "رنامج أصدقاء الدعم المبكر.xlsx"  
> **نوع الوحدة:** سيميوليشن محلي — بدون ربط بالباكند  
> **الموقع في dasm-investor:** تاب مستقل `/simulation` قابل للحذف بعد اكتمال الربط الحقيقي

---

## 1. المقدمة

### 1.1 الغرض
محاكاة كاملة لبرنامج أصدقاء الدعم المبكر — تتيح للمستخدم إدخال متغيرات المنصة (دخل، مساهمات، مصاريف) ومشاهدة توزيعات كل مشارك فورياً، واستكشاف سيناريوهات التخارج، مما يُمكّن من اتخاذ قرارات أفضل قبل تنفيذ البرنامج حقيقةً.

### 1.2 النطاق
- سيميوليشن **محلي بالكامل** (لا API، لا قاعدة بيانات)
- جميع الحسابات في الذاكرة (React state)
- قابل للتصدير (JSON + Excel وهمي)
- يُحذف أو يُخفى بعد اكتمال الربط مع الباكند الحقيقي

### 1.3 المصطلحات

| المصطلح | التعريف |
|---------|---------|
| **الصغار / الأبطال** | مشاركون بمساهمات أقل من 100,000 ر.س تراكمياً |
| **الكبار** | مشاركون تجاوزت مساهماتهم 100,000 ر.س تراكمياً |
| **إجمالي الدخل** | إجمالي العمولات قبل الخصومات (Gross Revenue) |
| **صافي الربح** | إجمالي الدخل − ضريبة − رسوم بوابة − عمولات شركاء |
| **نسبة المساهمة التراكمية** | مساهمة الشخص ÷ إجمالي المساهمات حتى تاريخ التوزيع |
| **2x / 3x / 4x** | مضاعف ما استلمه المشارك مقارنةً بما دفعه |
| **السقف** | عند 4x تنتهي حقوق المشارك المالية تلقائياً |
| **التخارج** | استرداد رأس المال فقط، لا بيع حصص ملكية |

## 2. وصف البرنامج

### 2.1 الفكرة الأساسية
برنامج تمويلي خاص **ليس بيع حصص ملكية** — المشاركون يحصلون على:
- **15% من إجمالي دخل المنصة** (للصغار) موزّعة بحسب نسبة مساهمتهم
- **نسبتهم من صافي الربح** (للكبار بعد تجاوز 100K)
- لا تدخل في القرارات، لا حق تصويت، لا ملكية في الأصول

### 2.2 الشرائح والعتبات

```
شريحة الأصدقاء (صغار):
  العتبة: 150,000 ر.س إجمالي تراكمي للمجموعة
  الوعاء: 15% من إجمالي الدخل الشهري
  الحد الأقصى للفرد: عند تجاوز 100,000 ر.س → يُرقّى للكبار

شريحة الملائكية الأولى (كبار):
  العتبة: 1,500,000 ر.س إقفال الشريحة
  الوعاء: نسبتهم من صافي الربح بعد المصاريف
  التوزيع: كل 3 أشهر
```

### 2.3 آلية السقف المتناقص

| مرحلة | ما استلمه | حصته |
|-------|-----------|------|
| المرحلة الأولى | 0x → 2x | 100% من حصته الأصلية |
| المرحلة الثانية | 2x → 3x | 50% من حصته الأصلية |
| المرحلة الثالثة | 3x → 4x | 25% من حصته الأصلية |
| ما بعد 4x | — | انتهت الحقوق تلقائياً |

### 2.4 نموذج دخل المنصة

```
الصفقات: فئة 250 ر.س + فئة 500 ر.س + فئة 1000 ر.س
إجمالي العمولات = Σ (عدد الصفقات × الرسوم)
إجمالي الدخل = العمولات × (1 + ضريبة 15%)   ← للتوزيع للصغار
صافي الدخل = إجمالي الدخل − ضريبة − رسوم بوابة (2.5%) − عمولات شركاء (20%)
```

## 3. المتطلبات الوظيفية

### FR-01: إدارة المشاركين
- **FR-01.1** يستطيع المستخدم إضافة مشارك جديد (اسم، مساهمات شهرية)
- **FR-01.2** يستطيع تعديل أو حذف مشارك
- **FR-01.3** النظام يصنّف تلقائياً: صغار / كبار بناءً على المساهمة التراكمية (< 100K / ≥ 100K)
- **FR-01.4** يُظهر نسبة المساهمة التراكمية لكل مشارك في كل شهر

### FR-02: نموذج الدخل
- **FR-02.1** يستطيع المستخدم إدخال عدد الصفقات شهرياً لكل فئة (250 / 500 / 1000 ر.س)
- **FR-02.2** النظام يحسب تلقائياً: إجمالي الدخل، الضريبة، رسوم البوابة، عمولات الشركاء، صافي الربح
- **FR-02.3** الأرقام قابلة للتعديل المباشر (Manual override)
- **FR-02.4** يدعم 3 سنوات (2026، 2027، 2028) + مرونة إضافة سنوات

### FR-03: حساب التوزيعات
- **FR-03.1** لا يبدأ توزيع الصغار حتى تبلغ المساهمات التراكمية 150,000 ر.س
- **FR-03.2** توزيع الصغار = 15% من إجمالي الدخل × نسبة المشارك التراكمية
- **FR-03.3** توزيع الكبار = نسبتهم × صافي الربح (بعد كسر 100K)
- **FR-03.4** النظام يتتبع الحالة التراكمية لكل مشارك: كم استلم، ما نسبة مضاعفه، أي مرحلة هو فيها
- **FR-03.5** عند وصول مشارك لـ 4x → يُوقف توزيعه تلقائياً مع رسالة واضحة

### FR-04: آلية المضاعف المتناقص
- **FR-04.1** بين 0x و 2x: المشارك يأخذ حصته الكاملة
- **FR-04.2** بين 2x و 3x: المشارك يأخذ 50% من حصته المحسوبة
- **FR-04.3** بين 3x و 4x: المشارك يأخذ 25% من حصته المحسوبة
- **FR-04.4** بعد 4x: صفر — ويُظهر النظام "اكتملت حقوقه المالية"

### FR-05: سيناريوهات التخارج
- **FR-05.1** سيناريو 1: تخارج نهاية 2026 → رأس المال فقط (لا أرباح إضافية)
- **FR-05.2** سيناريو 2: تخارج نهاية 2027 → رأس المال فقط + عرض ما استُلم من أرباح
- **FR-05.3** سيناريو 3: تخارج بعد 2028 → رأس المال فقط + عرض نسبة الربح / المساهمات
- **FR-05.4** التخارج مشروط: 3 سنوات حظر + 3 أشهر إشعار + الكاش المتوفر
- **FR-05.5** الكبار (دسام / أبو المعاطي / مدحت): ⛔ لا تخارج (حسب الملف)

### FR-06: لوحة التحكم والمقاييس
- **FR-06.1** عرض إجمالي المساهمات لكل مشارك (تراكمي)
- **FR-06.2** عرض إجمالي ما استلمه كل مشارك (تراكمي)
- **FR-06.3** عرض المضاعف الحالي لكل مشارك (استلم ÷ ساهم)
- **FR-06.4** شريط تقدم نحو السقف (4x)
- **FR-06.5** عرض متى يُتوقع بلوغ 2x / 3x / 4x بناءً على معدل التوزيع الحالي

### FR-07: التصدير
- **FR-07.1** تصدير حالة السيميوليشن كـ JSON
- **FR-07.2** تصدير جدول التوزيعات كـ CSV
- **FR-07.3** استيراد حالة محفوظة مسبقاً من JSON

| المتطلب | التفصيل |
|---------|---------|

| **اللغة** | عربي RTL كامل |
| **الاستجابة** | يعمل على desktop وtablet |

## 6. قواعد العمل الحرجة

```
1. الصغار لا يستلمون توزيعات حتى الإجمالي التراكمي يبلغ 150,000 ر.س
2. مجرد تجاوز 100,000 ر.س مساهمة فردية → الشخص يُنقل لشريحة الكبار
3. المضاعف يُحسب على مستوى الفرد، لا على مستوى المجموعة
4. التخارج = رأس المال فقط لا ربح إضافي
5. بعد 4x تنتهي الحقوق تلقائياً بدون إجراء
6. الكبار الثلاثة (من السيناريو: دسام، أبو المعاطي، مدحت) لا تخارج حسب الملف
7. التوزيع كل 3 أشهر (فصلي) — ليس شهرياً في الواقع، لكن الحساب شهري
```
