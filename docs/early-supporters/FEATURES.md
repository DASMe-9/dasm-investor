# FEATURES — برنامج أصدقاء الدعم المبكر (سيميوليشن)

> **الموقع:** `/simulation` تاب داخل `dasm-investor`  
> **النوع:** سيميوليشن محلي — بدون API — بيانات في الذاكرة

---

## Feature 1: إدارة المشاركين 👥

### الوصف
إضافة وتعديل وحذف المشاركين في البرنامج، مع تصنيف تلقائي لكل مشارك إلى شريحته الصحيحة.

### قصة المستخدم
> "كمدير للبرنامج، أريد أن أضيف أسماء المشاركين ومساهماتهم الشهرية، وأرى على الفور تصنيفهم وحصصهم."

### السلوك المتوقع

**إضافة مشارك:**
- حقول: الاسم، المساهمة الشهرية الافتراضية
- يمكن تخصيص مبلغ مختلف لكل شهر
- بعد الإضافة: يظهر في الجدول فوراً مع النسبة والتصنيف

**التصنيف التلقائي:**
```
المساهمة التراكمية < 100,000 ر.س  → 🟢 صغار / أبطال
المساهمة التراكمية ≥ 100,000 ر.س  → 🔵 كبار
```

**عرض بيانات المشارك:**
| الحقل | الوصف |
|-------|-------|
| الاسم | اسم المشارك |
| إجمالي مساهماته | الرقم التراكمي |
| نسبته من المجموعة | % حصته التراكمية |
| تصنيفه | صغار / كبار |
| وضعه من المضاعف | 1x → 2x → 3x → 4x ✅ |
| ما استلمه | الرقم التراكمي |

### حالات الخطأ
- اسم فارغ → رسالة تحذير
- مساهمة سالبة → غير مسموح
- حذف مشارك له توزيعات → تحذير تأكيد

## Feature 2: نموذج دخل المنصة 💰

### الوصف
إدخال بيانات دخل المنصة الشهرية بثلاث طرق: إدخال يدوي، أو حساب من الصفقات، أو استخدام بيانات الملف كأرقام افتراضية.

### قصة المستخدم
> "أريد أن أُدخل عدد الصفقات لكل فئة كل شهر وأرى كيف يتأثر توزيع المشاركين فوراً."

### الحسابات

```
إجمالي العمولات = (عدد 250 × 250) + (عدد 500 × 500) + (عدد 1000 × 1000)
ضريبة القيمة المضافة = إجمالي العمولات × (15 / 115)
إجمالي الدخل = إجمالي العمولات + ضريبة القيمة المضافة
رسوم بوابة الدفع = إجمالي الدخل × 2.45%
صافي للمنصة = إجمالي الدخل − رسوم البوابة
عمولات الشركاء = صافي للمنصة × 20%
صافي الربح = صافي للمنصة − عمولات الشركاء
```

### السلوك
- تعديل أي رقم → إعادة حساب فورية لكل التوزيعات
- Slider لكل فئة من الصفقات
- عرض الأرقام بالريال السعودي مع تنسيق (1,000,000 ر.س)
- عرض النسب المئوية لكل خصم
- زر "استخدم أرقام 2026 الفعلية" / "2027 التقديرية" / "2028 التقديرية"

### جدول الدخل الافتراضي (من الملف)

| السنة | إجمالي الدخل | صافي الربح |
|-------|-------------|-----------|
| 2026 | 328,000 ر.س | 255,975 ر.س |
| 2027 | 1,064,500 ر.س | 829,895 ر.س |
| 2028 | 2,450,000 ر.س | 1,910,224 ر.س |

## Feature 3: آلية التوزيع ثنائية الشريحة 📊

### الوصف
حساب حصة كل مشارك شهرياً بناءً على شريحته (صغار/كبار) وعتبة التفعيل.

### قصة المستخدم
> "أريد أن أرى كم سيحصل كل شخص هذا الشهر، مع مراعاة أن الصغار يأخذون من الإجمالي والكبار من الصافي."

### خوارزمية التوزيع

**الخطوة 1: هل تحقق الحد الأدنى للتفعيل؟**
```
إجمالي المساهمات التراكمية لكل الصغار ≥ 150,000 ر.س؟
  نعم → تفعيل التوزيع للصغار
  لا  → لا توزيع (عرض: "متبقي X ر.س لتفعيل التوزيع")
```

**الخطوة 2: حساب حصة الصغار**
```
وعاء الصغار = إجمالي الدخل × 15%
حصة الشخص = وعاء الصغار × (مساهمته التراكمية ÷ إجمالي مساهمات الصغار)
```

**الخطوة 3: تطبيق المضاعف المتناقص**
```
مضاعفه الحالي = (ما استلمه تراكمياً) ÷ (ما ساهم به تراكمياً)

if مضاعفه < 2.0:
    حصته الفعلية = حصته المحسوبة × 100%
elif مضاعفه < 3.0:
    حصته الفعلية = حصته المحسوبة × 50%
elif مضاعفه < 4.0:
    حصته الفعلية = حصته المحسوبة × 25%
else:
    حصته الفعلية = 0 (انتهت الحقوق)
```

**الخطوة 4: حصة الكبار**
```
وعاء الكبار = صافي الربح
حصة الشخص الكبير = وعاء الكبار × (مساهمته ÷ إجمالي مساهمات الكبار)
```

### عرض التوزيع
- جدول شهري: اسم | حصة الشهر | تراكمي | مضاعف | المرحلة
- تلوين تلقائي للمراحل: 🟢 أقل من 2x | 🟡 2x-3x | 🟠 3x-4x | ⛔ 4x+
- شريط تقدم نحو السقف لكل شخص

## Feature 4: سيناريوهات التخارج 🚪

### الوصف
عرض 3 سيناريوهات واضحة للتخارج مع الشروط والأرقام الفعلية لكل مشارك.

### قصة المستخدم
> "أريد أن أعرف بالضبط: إذا تخارج شخص نهاية 2027، كم يأخذ؟ وهل هذا مجدٍ له؟"

### السيناريوهات الثلاثة

**سيناريو 1: التخارج نهاية 2026**
```
قيمة التخارج = إجمالي مساهماته 2026 فقط (رأس المال)
ملاحظة: لا تخارج فعلي في 2026 — مدة الحظر 3 سنوات
```

**سيناريو 2: التخارج نهاية 2027**
```
قيمة التخارج = مساهمات 2026 + مساهمات 2027 (رأس المال الكامل)
معروض: إجمالي الأرباح المستلمة | نسبة الربح ÷ المساهمات
```

**سيناريو 3: التخارج بعد 2028**
```
قيمة التخارج = رأس المال التراكمي الكامل
معروض: إجمالي الأرباح 2026+2027+2028 | المضاعف المحقق | الحالة
```

### جدول عرض كل سيناريو

| الحقل | 2026 | 2027 | 2028 |
|-------|------|------|------|
| رأس المال | ✅ | ✅ | ✅ |
| الأرباح المستلمة | تُعرض | تُعرض | تُعرض |
| نسبة الربح / المساهمات | تُحسب | تُحسب | تُحسب |
| المضاعف المحقق | — | — | تُعرض |
| الحالة | لا تخارج | جزئي | بطل / كبير |

### شروط التخارج (تُعرض كـ Banner)
```
⏳ مدة الحظر: 3 سنوات من أول مساهمة
📢 إشعار مسبق: 3 أشهر
💰 يعتمد على الكاش المتوفر في المنصة
⛔ الكبار (> 100K مساهمة واحدة): لا تخارج
```

## Feature 6: محاكاة "ماذا لو" 🔮

### الوصف
تغيير متغير واحد ومشاهدة تأثيره على الجميع فوراً.

### قصة المستخدم
> "ماذا لو ارتفع دخل المنصة 20%؟ أو دخل مستثمر جديد بـ 200,000 ر.س؟"

### المتغيرات القابلة للتغيير
- نسبة نمو الدخل الشهري (slider: -50% → +200%)
- إضافة مستثمر كبير جديد وسط الفترة
- تغيير نسبة الـ 15% (للاستكشاف)
- تغيير عتبة التفعيل (150K)
- تغيير نسب المضاعف (2x/3x/4x)

### العرض
- جدول مقارنة: الوضع الحالي vs الوضع الجديد
- الفرق بالأرقام وبالنسبة المئوية
- خط زمني متوقع لوصول كل شخص للسقف

## Feature 7: التصدير والاستيراد 💾

### الوصف
حفظ حالة السيميوليشن واستعادتها لاحقاً أو مشاركتها.

### التصدير
- **JSON:** حالة كاملة (المشاركون + الدخل + التوزيعات) — قابل للاستيراد
- **CSV:** جدول التوزيعات الشهرية — لفتحه في Excel

### الاستيراد
- رفع ملف JSON محفوظ مسبقاً → استعادة الحالة كاملة
