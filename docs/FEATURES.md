## Feature 1: إدارة العقود

### الوصف
نظام إدارة العقود الاستثمارية يتيح للإدارة إنشاء عقود قانونية بأقسام ثابتة ومرنة، ونشر نسخ متعددة من كل عقد مع تتبع دقيق لأي نسخة وقّع عليها كل مستثمر.

### المستخدم المستهدف
- الأدمن / فريق العمليات القانونية في DASM

### قصص المستخدم

**قصة 1:** كأدمن، أريد إنشاء عقد استثماري جديد بعنوان ورقم مرجعي وسقف افتراضي، حتى أتمكن من توجيه مستثمرين جدد للتوقيع عليه.

**قصة 2:** كأدمن، أريد نشر نسخة جديدة من نص العقد عند تغيير الشروط، دون أن يتأثر المستثمرون الذين وقّعوا على النسخة القديمة.

**قصة 3:** كأدمن، أريد أرشفة عقد قديم حتى لا يُعرض على المستثمرين الجدد.

### السلوك المتوقع

- إنشاء عقد يبدأ بحالة `draft`
- الأدمن ينشر نسخة أولى (version=1) حتى يصبح العقد جاهزاً للاشتراكات
- كل نشر جديد يُنشئ سجلاً جديداً في `angel_investment_contract_versions`
- المستثمرون المرتبطون بنسخة قديمة لا يتأثرون بنشر نسخة جديدة
- `published_at` يُسجَّل فقط عند الضغط على زر "نشر" — المسودات تبقى بـ `published_at = null`

### القيود

- `contract_number` فريد على مستوى قاعدة البيانات — يرفض التكرار
- `version` رقمي تصاعدي داخل العقد الواحد — لا يمكن نشر نسخة برقم أقل من آخر نسخة منشورة
- `body_template` يدعم placeholders مثل `{{investor_name}}`, `{{principal_amount}}`, `{{cap_multiple}}`

### API Endpoints

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| GET | `/api/admin/angel-investments/contracts` | جلب جميع العقود |
| POST | `/api/admin/angel-investments/contracts` | إنشاء عقد جديد |
| POST | `/api/admin/angel-investments/contracts/{id}/versions` | نشر نسخة جديدة |

### حالات الخطأ

| الحالة | الكود | الرسالة |
|--------|-------|---------|
| `contract_number` مكرر | 422 | `contract_number already exists` |
| `version` رقم مُستخدم | 422 | `Version number already exists for this contract` |
| `body_template` فارغ | 422 | `body_template is required` |
| عقد غير موجود | 404 | `Contract not found` |

## Feature 2: المراكز الاستثمارية

### الوصف
نظام إنشاء وإدارة مراكز الاستثمار الفردية. كل مركز هو علاقة بين مستثمر محدد وعقد محدد بشروط مالية خاصة (رأس مال + سقف).

### المستخدم المستهدف
- الأدمن (إنشاء وإدارة)
- المستثمر (عرض فقط)

### قصص المستخدم

**قصة 1:** كأدمن، أريد إنشاء مركز استثماري لمستثمر معين بمبلغ 500,000 ريال ومضاعف 2×، حتى يحق له الحصول على عوائد حتى يستلم 1,000,000 ريال.

**قصة 2:** كمستثمر، أريد رؤية جميع مراكزي الاستثمارية مع تقدم كل مركز نحو سقفه.

**قصة 3:** كمستثمر، أريد رؤية تاريخ توقيعي وتاريخ التفعيل لكل مركز.

**قصة 4:** كأدمن، أريد إيقاف مركز مؤقتاً في حالات استثنائية دون إغلاقه نهائياً.

### السلوك المتوقع

- إنشاء مركز يبدأ دائماً بحالة `pending_signature`
- `subscription_code` يُولَّد تلقائياً بصيغة `ANGEL-XXXX-YYYY` (حيث XXXX عشوائي، YYYY سنة الإنشاء)
- الأدمن يختار النسخة الأحدث (المنشورة) من العقد تلقائياً عند الإنشاء
- لا يمكن تعديل `principal_amount` أو `cap_multiple` بعد إنشاء المركز

### القيود

- مستثمر واحد يمكن أن يملك أكثر من مركز في نفس العقد (لا unique constraint على (user_id, contract_id))
- `principal_amount` يجب أن يكون موجباً (> 0)
- `cap_multiple` يجب أن يكون ≥ 1.0

### حسابات المقاييس

```
cap_total = principal_amount × cap_multiple
paid_to_date = SUM(distributions.amount) WHERE subscription_id = this.id
remaining_to_cap = cap_total - paid_to_date
realized_multiple = paid_to_date / principal_amount
progress_percent = (paid_to_date / cap_total) × 100
```

### API Endpoints

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| GET | `/api/investor/angel-investments` | جميع مراكز المستثمر |
| GET | `/api/investor/angel-investments/{id}` | تفاصيل مركز |
| GET | `/api/investor/angel-investments/statements/portfolio` | ملخص المحفظة |
| POST | `/api/admin/angel-investments/subscriptions` | إنشاء مركز جديد |
| POST | `/api/admin/angel-investments/subscriptions/{id}/activate` | تفعيل |
| POST | `/api/admin/angel-investments/subscriptions/{id}/pause` | إيقاف مؤقت |
| POST | `/api/admin/angel-investments/subscriptions/{id}/resume` | استئناف |
| POST | `/api/admin/angel-investments/subscriptions/{id}/close` | إغلاق نهائي |

### حالات الخطأ

| الحالة | الكود | الرسالة |
|--------|-------|---------|
| مستثمر غير موجود | 404 | `Investor not found` |
| عقد غير موجود | 404 | `Contract not found` |
| لا توجد نسخة منشورة | 422 | `No published version for this contract` |
| تفعيل مركز لم يُوقَّع | 422 | `Subscription must be signed before activation` |
| إغلاق مركز مُغلق | 422 | `Subscription is already closed` |
| إيقاف مركز غير نشط | 422 | `Only active subscriptions can be paused` |

## Feature 3: التوقيع الإلكتروني

### الوصف
نظام توقيع إلكتروني آمن يتيح للمستثمر قراءة نص عقده كاملاً والموافقة عليه بشكل قانوني ملزم. يستخدم تجزئة SHA256 لضمان عدم إنكار المستثمر لمحتوى ما وقّع عليه.

### المستخدم المستهدف
- المستثمر (يوقّع مرة واحدة فقط لكل مركز)

### قصص المستخدم

**قصة 1:** كمستثمر، أريد قراءة نص عقدي الكامل قبل التوقيع، حتى أكون على علم كامل بالشروط والمخاطر.

**قصة 2:** كمستثمر، أريد التوقيع على العقد إلكترونياً من بوابتي، دون الحاجة لزيارة مكتب أو طباعة أوراق.

**قصة 3:** كأدمن، أريد التحقق من أن المستثمر وقّع على النص الصحيح وليس نسخة معدّلة.

### تفاصيل الأمان

#### حساب التجزئة
```
consent_anchor = نص الموافقة الثابت في الواجهة
hash = SHA256( version_id + "|" + user_id + "|" + consent_anchor )
```

**مثال:**
```
version_id = "3"
user_id = "142"
consent_anchor = "أوافق على شروط وأحكام عقد الاستثمار الملائكي..."
hash = SHA256("3|142|أوافق على شروط...")
     = "a7f3c9b2..."
```

#### البيانات المُسجَّلة
- `ip_address` — مأخوذ من `$request->ip()` في Laravel
- `user_agent` — مأخوذ من `$request->userAgent()`
- `accepted_at` — `now()` بتوقيت UTC
- `signer_name` — `$user->name` وقت التوقيع (لا يتغير لاحقاً)

### السلوك المتوقع

1. المستثمر يفتح صفحة عقده في البوابة
2. النظام يجلب نص العقد عبر `AngelInvestmentContractRenderer` مع استبدال placeholders
3. المستثمر يقرأ العقد ويضغط "أوافق وأوقّع"
4. الفرونتند يرسل `POST /{id}/sign` مع `{accepted: true}`
5. الباكند:
   - يتحقق أن `accepted = true`
   - يحسب `consent_text_hash`
   - يُنشئ سجل في `angel_investment_signatures`
   - يُحدّث حالة الاشتراك إلى `signed_pending_funding`
   - يُسجّل في `angel_investment_audit_logs`
6. المستثمر يرى رسالة تأكيد

### القيود

- توقيع واحد فقط لكل مركز — الجدول عليه `UNIQUE(subscription_id)`
- محاولة التوقيع مجدداً تُعيد خطأ `already_signed`
- التوقيع لا يُحذف ولا يُعدَّل — دائم

### API Endpoints

| الطريقة | المسار | السكوب | الوصف |
|---------|--------|--------|-------|
| GET | `/api/investor/angel-investments/{id}/contract` | `angel:read` | جلب نص العقد المُصيَّر |
| POST | `/api/investor/angel-investments/{id}/sign` | `angel:sign` | تنفيذ التوقيع |

### حالات الخطأ

| الحالة | الكود | الرسالة |
|--------|-------|---------|
| الاشتراك ليس في حالة `pending_signature` | 422 | `Subscription is not awaiting signature` |
| تم التوقيع مسبقاً | 422 | `Subscription already signed` |
| `accepted` = false | 422 | `Must explicitly accept the contract` |
| المستثمر لا يملك هذا المركز | 403 | `Forbidden` |

## Feature 4: التوزيعات وتتبع السقف

### الوصف
نظام تسجيل التوزيعات المالية للمستثمرين مع ربطها بالليدجر المحاسبي. يتتبع التقدم نحو السقف تلقائياً ويُغلق المركز عند الوصول.

### المستخدم المستهدف
- الأدمن (تسجيل التوزيعات)
- المستثمر (عرض التاريخ والتقدم)

### قصص المستخدم

**قصة 1:** كأدمن، أريد تسجيل دفعة شهرية لمستثمر بمبلغ 15,000 ريال، مع ربطها تلقائياً بالليدجر.

**قصة 2:** كأدمن، أريد أن يُغلق المركز تلقائياً عند وصول إجمالي التوزيعات للسقف، دون تدخل يدوي.

**قصة 3:** كمستثمر، أريد رؤية كل دفعة استلمتها مع تاريخها ووصفها وما تبقى حتى السقف.

### منطق الليدجر

```
عند إنشاء توزيع جديد:
  1. INSERT INTO ledger_entries (type='TYPE_ANGEL_DISTRIBUTION', amount, ...)
  2. INSERT INTO angel_investment_distributions (subscription_id, amount, ledger_entry_id, ...)
  3. احسب paid_to_date = SUM(distributions.amount) WHERE subscription_id = X
  4. إذا paid_to_date >= cap_total:
       UPDATE subscriptions SET status='capped' WHERE id = X
```

**قاعدة أساسية:** لا يُحذف أي إدخال من الليدجر — إذا كان هناك خطأ، يُُنشأ إدخال تصحيحي عكسي.

### السلوك المتوقع

- الأدمن يُدخل مبلغ التوزيع ووصفاً اختيارياً
- النظام يُنشئ إدخال الليدجر أولاً (atomic transaction)
- النظام يُنشئ سجل التوزيع مرتبطاً بـ `ledger_entry_id`
- إذا كان `paid_to_date + new_amount >= cap_total`:
  - يُسجَّل التوزيع بالمبلغ الكامل المُدخَل
  - حالة المركز تتغير إلى `capped` تلقائياً
  - لا يمكن تسجيل توزيعات إضافية بعد ذلك

### القيود

- لا توزيعات على مراكز `pending_signature`, `signed_pending_funding`, `capped`, `closed`
- مراكز `paused` لا تقبل توزيعات أثناء الإيقاف
- `amount` يجب أن يكون موجباً (> 0)
- العملة يجب أن تطابق عملة الاشتراك

### API Endpoints

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| POST | `/api/admin/angel-investments/subscriptions/{id}/distribute` | تسجيل توزيع |
| GET | `/api/investor/angel-investments/{id}/distributions` | عرض التوزيعات |
| GET | `/api/investor/angel-investments/{id}/ledger` | عرض الليدجر |

### حالات الخطأ

| الحالة | الكود | الرسالة |
|--------|-------|---------|
| مركز غير نشط | 422 | `Subscription is not active` |
| مبلغ سالب أو صفر | 422 | `Amount must be greater than zero` |
| عملة غير مطابقة | 422 | `Currency mismatch` |
| المركز وصل للسقف | 422 | `Subscription has reached its cap` |

## Feature 5: لوحة المستثمر

### الوصف
واجهة المستثمر الكاملة في `investor.dasm.com.sa` — مبنية بـ React + Vite — تعرض المحفظة الاستثمارية وتتيح التوقيع على العقود ومتابعة التوزيعات.

### الصفحات المتاحة

#### صفحة الداشبورد (/)
**البيانات المعروضة:**
- إجمالي رأس المال المستثمر (مجموع كل المراكز)
- إجمالي العوائد المستلمة
- إجمالي المتبقي حتى السقف
- عدد المراكز النشطة
- بطاقات ملخص لكل مركز: الكود، الحالة، شريط تقدم السقف

#### صفحة المحفظة (/portfolio)
**البيانات المعروضة:**
- جدول تفصيلي لجميع المراكز
- لكل مركز: subscription_code، العقد، رأس المال، السقف، المدفوع، نسبة التقدم، الحالة
- فلترة حسب الحالة
- رابط للتفاصيل

#### صفحة تفاصيل المركز (/subscription/:id)
**البيانات المعروضة:**
- معلومات المركز الكاملة
- تاريخ التوقيع وتاريخ التفعيل
- جدول التوزيعات
- زر "عرض العقد" إذا كان المستثمر لم يوقّع بعد

#### صفحة العقد (/subscription/:id/contract)
**البيانات المعروضة:**
- نص العقد كاملاً (HTML مُصيَّر من الباكند)
- نص الإفصاح عن المخاطر
- زر "أوافق وأوقّع" — يظهر فقط إذا الحالة = `pending_signature`
- بعد التوقيع: رسالة تأكيد مع تاريخ التوقيع

### تدفق المستثمر الجديد

```
تسجيل الدخول → SSO redirect → الداشبورد
  ↓ (إذا كان هناك مركز بانتظار التوقيع)
إشعار "لديك عقد بانتظار توقيعك"
  ↓
صفحة العقد → قراءة → توقيع
  ↓
رسالة "شكراً — سيتواصل معك الفريق لتأكيد التمويل"
```

## Feature 6: حساب العوائد 15%

### الوصف
خدمة `InvestorRevenueCalculator` تحسب حصة المستثمرين من صافي دخل المنصة اليومي بنسبة 15%، وتوزّع هذا المبلغ على المراكز النشطة بما يتناسب مع رأس مال كل مركز.

### صيغة الحساب

```
صافي_دخل_المنصة = إجمالي_إيرادات_المنصة - إجمالي_مصاريف_التشغيل

حصة_المستثمرين = صافي_دخل_المنصة × 0.15

لكل مركز نشط:
  وزن_المركز = principal_amount / إجمالي_رأس_المال_النشط
  حصة_المركز = حصة_المستثمرين × وزن_المركز
```

### مثال تطبيقي

```
صافي دخل المنصة اليومي = 100,000 ريال
حصة المستثمرين (15%) = 15,000 ريال

المراكز النشطة:
  مركز A: رأس مال 500,000 → وزن 50%  → حصة 7,500 ريال
  مركز B: رأس مال 300,000 → وزن 30%  → حصة 4,500 ريال
  مركز C: رأس مال 200,000 → وزن 20%  → حصة 3,000 ريال
```

### المتغيرات

| المتغير | المصدر | الوصف |
|---------|--------|-------|
| `net_platform_income` | قاعدة البيانات / تقرير يومي | صافي الدخل للفترة |
| `active_subscriptions` | `angel_investment_subscriptions` WHERE `status='active'` | المراكز النشطة |
| `total_active_principal` | SUM(principal_amount) على المراكز النشطة | إجمالي رأس المال |
| `per_subscription_share` | `principal_amount / total_active_principal × 15%` | حصة كل مركز |

### مصادر البيانات

- **صافي الدخل:** يُحدَّد يدوياً من الأدمن عند تسجيل التوزيع (لا يوجد حساب تلقائي مباشر من الإيرادات حالياً)
- **المراكز النشطة:** استعلام مباشر على `angel_investment_subscriptions`

### ملاحظات مهمة

- الخدمة تحسب الأرقام فقط — لا تُسجِّل التوزيعات تلقائياً
- الأدمن يراجع الأرقام المقترحة ويؤكد التوزيع يدوياً
- هذا يضمن تدقيقاً بشرياً قبل أي حركة مالية

## Feature 7: نظام SSO للبوابة

### الوصف
نظام Single Sign-On يتيح للمستثمر الانتقال من `dasm.com.sa` إلى `investor.dasm.com.sa` بتوقيع واحد، مع إصدار توكن محدود الصلاحيات للبوابة.

### تدفق SSO الكامل

```
1. المستثمر مُسجَّل دخوله على dasm.com.sa
2. يضغط "بوابة المستثمر" في القائمة
3. الفرونتند (dasm.com.sa):
   POST /api/sso/generate
   Body: { platform: "investor", return_url: "https://investor.dasm.com.sa/dashboard" }
   Headers: Authorization: Bearer {main_access_token}

4. الباكند يتحقق:
   - المستخدم مُوثَّق؟ ✓
   - يملك دور "investor"؟ ✓
   - منحه scopes: [angel:read, angel:sign, angel:ledger]

5. الباكند يُنشئ SSO token (JWT):
   - expires: 5 دقائق
   - single_use: true
   - scopes: [angel:read, angel:sign, angel:ledger]
   - user_id: X

6. الباكند يُعيد:
   { sso_token: "eyJ...", expires_in: 300 }

7. الفرونتند يُعيد توجيه المتصفح:
   → https://investor.dasm.com.sa/auth/callback?sso_token=eyJ...

8. بوابة المستثمر (investor.dasm.com.sa):
   POST /api/sso/verify
   Body: { sso_token: "eyJ...", platform: "investor" }

9. الباكند:
   - يتحقق من صحة التوكن
   - يتحقق من عدم استخدامه مسبقاً
   - يُبطل التوكن (لا يمكن إعادة استخدامه)
   - يُصدر Sanctum Personal Access Token (30 يوم) بالسكوبات المحددة

10. البوابة تستلم:
    { access_token: "...", user: { id, name, email } }

11. البوابة تخزن access_token في localStorage
12. البوابة تُوجّه إلى الداشبورد
```

### حالات الخطأ وكيفية التعامل معها

| الحالة | السلوك |
|--------|--------|
| SSO token منتهي (5 دقائق مضت) | إعادة توجيه لـ dasm.com.sa مع رسالة "انتهت الجلسة — حاول مجدداً" |
| SSO token مستخدم مسبقاً | تسجيل في الـ audit log + إعادة للرئيسية مع رسالة خطأ |
| المستخدم فقد دور investor | 403 + إعادة للرئيسية |
| Access token منتهي (30 يوم) | الصفحة تُوجّه لـ /auth/login في البوابة، التي تُوجّه تلقائياً لـ SSO |

### السكوبات المتاحة للبوابة

```
angel:read   → قراءة المحفظة، العقود، التوزيعات
angel:sign   → تنفيذ التوقيع الإلكتروني
angel:ledger → عرض حركات الليدجر
```

**مهم:** هذه السكوبات لا تُتيح الوصول لـ APIs أخرى (سيارات، مزادات، إلخ).

### مدة التوكنات

| النوع | المدة | السبب |
|-------|-------|-------|
| SSO Token | 5 دقائق | أمان — يُستخدم مرة واحدة فقط |
| Access Token | 30 يوماً | تجربة مستخدم — لا يحتاج إعادة دخول متكررة |

## Feature 8: لوحة الإدارة

### الوصف
لوحة تحكم شاملة للأدمن في `control.dasm.com.sa` لإدارة جميع جوانب نظام الاستثمار الملائكي.

### الإجراءات المتاحة للأدمن

#### إدارة المستثمرين
- **عرض قائمة المستثمرين:** اسم، بريد، عدد المراكز، إجمالي استثماراتهم
- **ترقية مستخدم:** البحث عن مستخدم عادي ومنحه دور `investor`
- **عرض مراكز مستثمر محدد:** جميع اشتراكاته مع تفاصيل كل مركز

#### إدارة العقود
- **إنشاء عقد جديد:** نموذج بالحقول الأساسية
- **نشر نسخة جديدة:** محرر نصوص مع معاينة قبل النشر
- **عرض نسخ عقد محدد:** تاريخ جميع النسخ ومن وقّع على كل نسخة

#### إدارة الاشتراكات
- **إنشاء اشتراك:** ربط مستثمر بعقد بشروط محددة
- **تفعيل اشتراك:** بعد تأكيد استلام التمويل
- **تسجيل توزيع:** إدخال المبلغ والوصف
- **إيقاف / استئناف:** للمراكز النشطة
- **إغلاق نهائي:** مع تأكيد مزدوج (لا رجعة)

#### التقارير والمتابعة
- **نظرة عامة (Overview):** إجماليات سريعة
- **سجل التدقيق:** جميع الإجراءات الإدارية مع التواريخ والمنفّذين
- **تقارير مالية:** إجمالي التوزيعات، المراكز المكتملة، المراكز النشطة

### قيود الأدمن

- الأدمن لا يستطيع تعديل توقيع موجود
- الأدمن لا يستطيع حذف إدخالات الليدجر
- الأدمن لا يستطيع التراجع عن الإغلاق (`closed` → نهائي)
- الأدمن لا يستطيع تعديل `principal_amount` بعد الإنشاء
